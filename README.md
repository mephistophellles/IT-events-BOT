# IT-events-BOT

Telegram bot для отслеживания событий для предпринимателей и стартапов. Бот автоматически мониторит добавленные администратором ресурсы (каналы, блоги, сайты) на предмет новых событий, отправляет рассылки подписчикам и напоминания за день и час до начала события.

## Возможности

- 🔍 Автоматический мониторинг ресурсов (каналы, блоги, сайты)
- 🤖 Нейросетевая классификация событий для фильтрации релевантных событий
- 📢 Рассылка новых событий всем подписчикам
- 📅 Регистрация пользователей на события
- ⏰ Автоматические напоминания за день и за час до события
- 👥 Административная панель для управления ресурсами

## Установка

1. Клонируйте репозиторий или скачайте файлы проекта

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

   **Примечание для локальной модели:** Если планируете использовать локальную модель классификации (без OpenAI API), рекомендуется установить CPU-версию PyTorch для экономии места:
   ```bash
   pip install torch --index-url https://download.pytorch.org/whl/cpu
   ```
   Или если у вас есть GPU с CUDA:
   ```bash
   pip install torch --index-url https://download.pytorch.org/whl/cu118
   ```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

4. Получите API-ключ Gemini:
   - Откройте https://aistudio.google.com/app/apikey
   - Создайте ключ и сохраните его (он понадобится для `.env`)
   - Убедитесь, что у ключа есть доступ к модели `gemini-1.5-flash` (или другой выбранной модели)

5. Заполните `.env` файл:
   - `BOT_TOKEN` - токен вашего Telegram бота (получите у @BotFather)
   - `ADMIN_IDS` - ID администраторов через запятую (можно узнать у @userinfobot)
   - `USE_GEMINI_CLASSIFIER` - использовать Gemini для классификации (true/false, по умолчанию true)
   - `GEMINI_API_KEY` - API ключ Gemini
   - `GEMINI_MODEL` - название модели Gemini (по умолчанию `gemini-1.5-flash`)
   - `CLASSIFIER_CONFIDENCE_THRESHOLD` - порог уверенности классификатора (0.0-1.0, по умолчанию 0.4)

6. Запустите бота:
```bash
python bot.py
```

## Использование

### Команды для пользователей

- `/start` - Начать работу с ботом
- `/help` - Справка по командам
- `/events` - Список предстоящих событий
- `/my_events` - Список событий, на которые вы зарегистрированы
- `/subscribe` - Подписаться на рассылку новых событий
- `/unsubscribe` - Отписаться от рассылки

### Команды для администраторов

- `/admin` - Административная панель
- `/add_resource <название> <URL> <тип>` - Добавить ресурс для мониторинга
  - Типы: `channel`, `blog`, `website`
  - Пример: `/add_resource Startup Events https://example.com/events blog`
- `/list_resources` - Список всех добавленных ресурсов
- `/check_resources` - Вручную проверить ресурсы на новые события

## База данных

Бот использует SQLite базу данных (по умолчанию `events_bot.db`). База данных создается автоматически при первом запуске.

Таблицы:
- `resources` - Ресурсы для мониторинга
- `events` - Обнаруженные события
- `users` - Пользователи бота
- `event_registrations` - Регистрации пользователей на события
- `admins` - Администраторы

## Особенности

- Автоматическая проверка ресурсов на новые события (настраивается через `CHECK_INTERVAL_MINUTES`)
- **Нейросетевая классификация событий** - умная фильтрация событий с помощью ИИ:
  - Поддержка Gemini (Google) для точной классификации онлайн
  - Локальная модель (BART zero-shot) как альтернатива
  - Автоматический fallback на ключевые слова при недоступности моделей
  - Фильтрация нерелевантных событий (например, конференций для дизайнеров)
- Умное определение дат событий из текста
- Автоматические напоминания за 24 часа и за 1 час до события
- Защита от дублирования событий
- Поддержка различных типов ресурсов

## Классификация событий

Бот использует нейросетевую классификацию для определения релевантности событий. Это позволяет избежать попадания в рассылку событий, не связанных с предпринимательством и стартапами.

### Режимы работы классификатора:

1. **Gemini (Google)** — основной режим
   - Самая точная классификация
   - Требует интернет и API-ключ
   - Поддерживает русский и английский текст

2. **Локальная модель (BART)** — работает без внешних API
   - Автоматически загружается при недоступности Gemini
   - Использует BART zero-shot классификатор
   - Требует больше ресурсов (~1.5GB), но работает офлайн

3. **Ключевые слова** - fallback режим
   - Используется при недоступности моделей
   - Базовая фильтрация по ключевым словам

### Настройка порога уверенности:

Параметр `CLASSIFIER_CONFIDENCE_THRESHOLD` определяет минимальную уверенность классификатора для принятия события:
- `0.5` - более мягкая фильтрация (больше событий)
- `0.6` - сбалансированный режим (по умолчанию)
- `0.7-0.8` - строгая фильтрация (только явно релевантные события)

## Примечания

- Для доступа к Gemini требуется стабильный интернет и действующий API-ключ
- Следите за квотами Google AI Studio, чтобы бот не превышал бесплатный лимит
- Для мониторинга Telegram каналов потребуется дополнительная настройка (использование Telethon или Telegram API)
- Парсер событий использует эвристические методы и может требовать доработки под конкретные ресурсы
- Рекомендуется регулярно проверять работу парсера и при необходимости настраивать его под специфику ваших ресурсов
- При первом запуске с локальной моделью BART будет загружена модель (~1.5GB), это может занять время
- Если Gemini недоступен, бот автоматически переключится на локальную модель BART или ключевые слова
